Региональный правовой портал Чеченской Республики
Обзор
Региональный правовой портал — веб-приложение на базе FastAPI, обеспечивающее централизованный доступ к официальным документам министерств и ведомств Чеченской Республики. Система агрегирует данные из восьми источников, предоставляя единую точку входа для поиска и просмотра правовых актов, постановлений, приказов и поручений.

Проект включает автоматизированный парсер для сбора документов, поисковый движок с поддержкой фильтрации по категориям, а также интеллектуального ассистента на основе ИИ для обработки запросов на естественном языке.

Установка и запуск
Требования
Python 3.10+

pip

Установка
Клонирование репозитория:

bash
git clone <repository-url>
cd project
Создание и активация виртуального окружения:

bash
python -m venv venv
source venv/bin/activate      # Linux/Mac
venv\Scripts\activate         # Windows
Установка зависимостей:

bash
pip install -r requirements.txt
Настройка переменных окружения (файл .env):

env
# Cloudflare AI (опционально)
CLOUDFLARE_API_KEY=<api-key>
CLOUDFLARE_ACCOUNT_ID=<account-id>

# Режим отладки
DEBUG=True
Запуск
Запуск веб-сервера:

bash
python main.py
Приложение будет доступно по адресу: http://127.0.0.1:8000

Первичный сбор документов:

bash
python run_parser.py --all
Автоматическое обновление (еженедельно):

bash
python scheduler.py
Функциональность
Веб-приложение
Главная страница: список доступных источников с количеством документов

Просмотр документов: фильтрация по категориям, пагинация, прямой доступ к файлам

Поиск: поиск по названию и категориям (для поддерживаемых источников)

Управление парсером: запуск обновления через веб-интерфейс с отображением статуса

ИИ-ассистент: обработка запросов на естественном языке с поиском по всем базам данных

API Endpoints
Метод	Эндпоинт	Описание

GET	/api/ai/search	Поиск по базам данных
Парсер
Единый парсер поддерживает:

Обход всех страниц с пагинацией

Обработку защитных механизмов (cookie, редиректы)

Добавление только новых документов (исключение дубликатов)

Обновление как всех источников, так и выбранных

Команды парсера:

bash
# Обновить все источники
python run_parser.py --all

# Обновить конкретный источник
python run_parser.py --source vesti095

# Показать список источников
python run_parser.py --list
Структура баз данных
Каждая база данных SQLite содержит таблицу со следующей структурой:

Поле	Тип	Описание
id	INTEGER	Первичный ключ
title	TEXT	Название документа
date / pub_date / sign_date	TEXT	Дата (зависит от источника)
file_url	TEXT	Ссылка на файл
file_name	TEXT	Имя файла
category	TEXT	Категория документа
source_url	TEXT	URL источника
scraped_at	TIMESTAMP	Время добавления
Безопасность
CORS настроен с широкими разрешениями для разработки (рекомендуется ограничить в продакшене)

Базы данных SQLite хранятся локально и недоступны извне

API-ключи Cloudflare хранятся в .env и не включаются в репозиторий

Разработка
Добавление нового источника
Добавить конфигурацию в Config.SOURCES в unified_parser.py

Реализовать функцию парсинга для нового источника

Добавить эндпоинт в main.py (при необходимости)

Расширение ИИ-контекста
Для улучшения ответов ИИ-ассистента можно расширить функцию search_all_databases() в main.py, добавив дополнительные источники данных.
