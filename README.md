Региональный правовой портал Чеченской Республики
Региональный правовой портал представляет собой комплексную информационную систему для централизованного хранения, поиска и анализа официальных документов федерального и регионального уровней. Система обеспечивает автоматический сбор документов из открытых источников, интеграцию с официальным API правовой информации Российской Федерации, интеллектуальный поиск с использованием семантического анализа и взаимодействие с пользователем через веб-интерфейс.

Архитектура системы
Система построена по модульному принципу с четким разделением ответственности между компонентами. В основе лежит веб-фреймворк FastAPI, обеспечивающий высокую производительность и асинхронную обработку запросов. Архитектура включает два независимых контура обработки данных: региональный (сбор с сайтов ведомств ЧР) и федеральный (интеграция с API pravo.gov.ru).

Функциональные возможности
Основные функции
1. Централизованное хранение документов

Система объединяет документы из двух основных источников:

Региональный контур: 8 баз данных от министерств и ведомств Чеченской Республики

Федеральный контур: Интеграция с официальным API pravo.gov.ru для доступа к федеральным правовым актам

2. Интеллектуальный поиск

Реализован многоуровневый поиск с ранжированием результатов:

Поиск по всем текстовым полям (название, категория, тип, номер)

Использование синонимов для расширения поискового запроса

Вычисление релевантности на основе частоты вхождений и важности полей

Сортировка результатов по степени соответствия запросу

Поиск по федеральным документам с фильтрацией по блокам публикации, видам документов и принявшим органам

3. ИИ-ассистент

Интеграция с большой языковой моделью для обработки естественно-языковых запросов:

Анализ пользовательского запроса и извлечение ключевых понятий

Поиск релевантных документов во всех базах данных

Формирование структурированного ответа с перечнем найденных документов

Указание источников и ссылок для каждого документа

4. Автоматический сбор региональных документов

Единый парсер выполняет обход официальных сайтов ведомств ЧР:

Поддержка различных форматов страниц и структур данных

Обработка пагинации и динамической загрузки контента

Защита от дублирования документов

Фоновый режим работы с возможностью мониторинга статуса

5. Интеграция с федеральным API

Прямой доступ к федеральным правовым актам через официальный API:

Поиск документов по множеству параметров

Получение детальной информации о документе

Скачивание PDF-файлов

Получение справочных данных (блоки публикации, виды документов, принявшие органы)

Статистика публикаций по периодам

6. Фильтрация и навигация

Пользователь может фильтровать документы по:

Категории (предопределенные списки для Минздрава и Минобра)

Поисковому запросу в названии

Блоку публикации (федеральные документы)

Виду документа

Принявшему органу

Периоду публикации

Дате подписания

Номеру документа

Пагинация для удобного просмотра больших списков

Федеральные документы (кэш API)
Федеральные документы не хранятся локально, а получаются в реальном времени через API pravo.gov.ru. Для оптимизации производительности используется кэширование ответов API.

Клиент API (services/pravo_api.py)
Клиент реализует полный набор методов для взаимодействия с официальным API портала правовой информации.

Развертывание
Запуск приложения
bash
python main.py
Приложение будет доступно по адресу: http://127.0.0.1:8000

Запуск в production режиме
bash
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
Запуск парсера из командной строки
bash
# Обновление всех источников
python run_parser.py --all

# Обновление конкретного источника
python run_parser.py --source vesti095

# Список доступных источников
python run_parser.py --list
Запуск планировщика
bash
python scheduler.py
Планировщик будет запускать обновление региональных баз данных каждое воскресенье в 3:00.

Docker-развертывание
Создайте Dockerfile:

dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
Сборка и запуск:

bash
docker build -t legal-portal .
docker run -p 8000:8000 --env-file .env legal-portal
